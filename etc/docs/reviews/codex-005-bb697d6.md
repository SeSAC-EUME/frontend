커밋: bb697d6 (005: 관리자 페이지 STORAGE_KEYS 적용 및 API 연동)

- 문제: Users.jsx 데이터 로드 시 API 응답 구조를 가정(`response.users || response || []`)하지만 키 불일치 대응이 없습니다. 최소한 빈 배열/배열 타입 검증 후 매핑을 추가하고, 요소 필수 필드(id/name/email 등)가 없을 때를 대비한 가드가 필요합니다. 현재는 객체/배열 혼동 시 UI에서 런타임 오류가 날 수 있습니다.
- 문제: Users.jsx는 API 실패 시 샘플 데이터로 폴백하지만 로딩 상태 해제 이후에도 샘플/실데이터 구분 없이 표시합니다. 이는 의도지만, 실패 여부를 사용자에게 알리지 않아 운영 환경에서 조용히 샘플이 렌더될 위험이 있습니다. 최소한 콘솔 로그 외 사용자/관리자용 알림(토스트/에러 메시지) 또는 "실시간 데이터를 불러오지 못했습니다" 안내가 필요합니다.
- 제안: AdminLayout 세션 만료 처리 후에 `STORAGE_KEYS.ADMIN_LOGIN_HISTORY`도 함께 정리하면 로그인 히스토리도 일관되게 초기화됩니다(로그아웃에서는 삭제하지만 만료 시에는 남습니다).
